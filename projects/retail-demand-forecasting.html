1. Contexte & Enjeux (Business Case)

Le Défi : Le Dilemme du Stock

Dans la grande distribution, chaque responsable de magasin doit répondre chaque semaine à une question critique : "Combien dois-je commander ?" ...

  • La Rupture de Stock (Understock) : Ventes perdues, clients insatisfaits, image de marque dégradée.
  • Le Surstock (Overstock) : Trésorerie immobilisée, coûts de stockage, risques de perte (périssables) ou de démarque inconnue.

L'Environnement : Corporación Favorita (Équateur)

  • Volatilité promotionnelle : Les promotions massives créent des pics de demande soudains.
  • Calendrier complexe : Jours fériés locaux, régionaux et nationaux, souvent décalés.
  • Facteurs macro-économiques : Sensibilité aux prix du pétrole.
  • Diversité des produits : Des milliers de références, des produits de base aux produits de niche.

L'Objectif

  • Prévoir la demande hebdomadaire sur un horizon de 8 semaines.
  • Quantifier l'incertitude de cette prévision.
  • Recommander une quantité de commande optimale basée sur les coûts économiques.

2. Architecture & Stratégie de Données (ETL)

Transformation & Qualité (Python)

  • Grille Cartésienne : Génération des lignes manquantes pour distinguer "Pas de vente" et "Pas de demande".
  • Agrégation Hebdomadaire ISO : Utilisation du standard ISO 8601.
  • Sortie Parquet : Données nettoyées stockées dans data/processed/*.parquet.

Suivi des Expériences (Experiment Tracking)

  • forecasts.sqlite : Prévisions probabilistes et registre des runs.
  • metrics.sqlite : Erreurs et monitoring de drift.
  • decisions.sqlite : Recommandations de commandes optimisées.

3. Stratégie de Modélisation (Intelligence Artificielle)

Approche "Baseline Hybride" (Smart Routing)

La "Baseline" du projet n'est pas un modèle unique, c'est une Stratégie Hybride Intelligente (Smart Routing)...

  • Le Diagnostic (Analyse du Produit) : Calcul des indicateurs ADI et CV².
  • La Classification (Matrice Syntetos-Boylan) : Classement des produits selon ADI et CV².
  • Le "Routing" (Le Choix du Modèle) : Application de la meilleure formule pour chaque profil.
Type de Produit La Recette Appliquée Pourquoi ce choix ?
Produits Stables (Smooth) Mix (70% Moyenne + 30% Saisonnier) On lisse le bruit tout en gardant une légère mémoire de la saisonnalité.
Produits Nerveux (Erratic) Mix (50% Moyenne + 50% Saisonnier) La volatilité est forte : on fait plus confiance à la saisonnalité.
Produits Rares (Intermittent/Lumpy) Moyenne Mobile Robuste On assure le "minimum vital" pour éviter le surstockage.

La baseline cherche à être statistiquement robuste, évitant les erreurs classiques de modélisation.

4. Moteur de Décision (Inventory Optimization)

Prévision Probabiliste

  • P10 (Scénario Bas) : On est sûr à 90% de vendre au moins ça.
  • P50 (Médiane) : Le scénario le plus probable.
  • P90 (Scénario Haut) : On a 10% de chance de vendre plus que ça.

Logique "Newsvendor" (Vendeur de Journaux)

  • Si le coût de la rupture est élevé, le système cible un quantile élevé (ex: P80).
  • Si le coût de surstock est élevé, le système cible un quantile bas (ex: P40).

5. Guide Technique

Prérequis

  • Python 3.8+
  • Pip

Installation

git clone https://github.com/BeediGoua/retail-demand-forecasting-assistant
cd retail-demand-forecasting-assistant
pip install -r requirements.txt

Téléchargement des Données

pip install kaggle
kaggle competitions download -c store-sales-time-series-forecasting
unzip store-sales-time-series-forecasting.zip -d data/raw/

Construction des Données (Pipeline)

# 1. Traitement et Nettoyage
python scripts/preprocessing.py
# 2. Construction de la Base SQL
python scripts/build_warehouse.py
# 3. Vérification de l'Intégrité
audit python scripts/verify_warehouse_integrity.py

Lancement de l'Application

streamlit run app/Home.py
